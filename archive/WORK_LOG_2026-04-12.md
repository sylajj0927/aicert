## Frontend Bug Fix ‚Äî Card text and pronunciation disappeared

### Symptom
- Concept cards still showed category / difficulty / course tags
- But main concept text became empty
- Pronunciation / play-audio control also disappeared

### Root cause
`concepts.json` schema drifted from:
- `termEn` / `termZh`

to:
- `englishTerm` / `chineseTerm`

The frontend rendering logic in `App.tsx` was still reading `termEn` / `termZh`, so card title/text and pronunciation lost their data source, while tags still rendered because their fields were unaffected.

### Fix
Added a small normalization layer in `App.tsx` when loading `conceptsRaw`:
- `termEn = termEn ?? englishTerm ?? ''`
- `termZh = termZh ?? chineseTerm ?? ''`

Also kept the existing safety behavior:
- rows with blank `conceptId` remain excluded from rendering

### Result
- card title text restored
- English / Chinese concept text restored
- pronunciation button restored
- expand / collapse still works
- filtering still works
- no layout or scope changes
